doRNG 패키지

January 28, 2021 in R

R에서 느린 for 문의 계산속도를 높이는 방법 중 하나가 doParallel 패키지의 foreach()를 사용하는 것인데요.

하지만 reproduciblility를 위해서 foreach()를 사용하는 것은 주의해합니다. 보통 seed를 설정할 때 사용하는 set.seed()가 제대로 먹히지 않기 때문입니다.

다음의 예를 확인해볼까요?

library(doParallel)
cl <- makeCluster(2)
registerDoParallel(cl)

# foreach에서 %dopar% 구문으로 생성
set.seed(1234)
s1 <- foreach(i = 1:4) %dopar% { 
  runif(1) 
}

set.seed(1234)
s2 <- foreach(i = 1:4) %dopar% { 
  runif(1) 
}

# 두 벡터가 같은지 비교
identical(s1, s2)
## [1] FALSE

이와 같이 같은 seed를 주었음에도 생성된 두 벡터가 다르다는 것을 확인할 수 있습니다.

이는 doRNG 패키지를 활용하여 해결할 수 있습니다.!!


doRNG 패키지

1. %dorng%

방법은 다음과 같이 .options.RNG에 seed를 주고 %dopar%대신 %dorng%를 사용하는 것입니다.

library(doRNG)

# %dorng% 구문으로 생성
r1 <- foreach(i = 1:4, .options.RNG = 1234) %dorng% { 
  runif(1) 
}

r2 <- foreach(i = 1:4, .options.RNG = 1234) %dorng% { 
  runif(1) 
}
identical(r1, r2)
## [1] TRUE

처음의 결과와 달리 생성된 두 벡터가 같은 것을 확인할 수 있습니다.

2. registerDoRNG

또 다른 방법은 다음과 같습니다.

# %dorng% 구문으로 생성
r1 <- foreach(i = 1:4, .options.RNG = 1234) %dorng% { 
  runif(1) 
}

# regi
registerDoRNG(1234)
r2 <- foreach(i = 1:4) %dopar% { 
  runif(1) 
}

identical(r1, r2)
## [1] TRUE

이 방법 또한 같은 결과를 보여주는 것을 확인할 수 있습니다.

# stop cluster
stopCluster(cl)
